# Security Policy

## Reporting a Vulnerability
Please report security issues privately. You can:
- Open a confidential issue marked `security` (do not include exploit details)
- Or email the maintainer if listed in the repository profile

We will triage within a reasonable timeframe and provide fixes publicly once validated.

## Scope
- Source code in this repository
- Build workflows (GitHub Actions)

## Do Not
- Publicly disclose zero-day vulnerabilities before a fix is available
- Commit secrets or credentials in the repo

## Responsible Disclosure
We appreciate responsible disclosure and will credit reporters in release notes if requested.
